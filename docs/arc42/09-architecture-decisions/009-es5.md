# ADR-009: ES5 throughout (no transpilation)

## Status
Accepted

## Context
Modern JavaScript (ES6+) offers `const`, `let`, arrow functions, classes, template literals, destructuring, and modules. Using them would require a build step for older browser support.

## Decision
Write all framework code in ES5. `var`, `function`, `for` loops. No transpilation, no Babel, no build step.

## Rationale
- **"Open the HTML file" promise** — the framework works by including a `<script>` tag. No npm install, no webpack, no build. This is a core value proposition.
- **Maximum browser compatibility** — ES5 works everywhere, including embedded webviews and older enterprise browsers.
- **Consistency** — the engine runs in both browser and Node. ES5 is the lowest common denominator that works in both without transformation.
- **Simplicity** — no `.babelrc`, no `tsconfig.json`, no `webpack.config.js`. The absence of tooling config is the feature.

## Consequences
- No `const`/`let` — `var` is function-scoped, which requires care with closures in loops. IIFEs are used where needed.
- No arrow functions — `function` keyword everywhere. More verbose but unambiguous.
- No destructuring — `var x = obj.x` instead of `var { x } = obj`.
- No modules — UMD wrapper for compatibility. `require()` and `<script>` both work.
- Contributors must follow ES5 conventions (documented in CONTRIBUTING.md).
