What this demonstrates

  • Compound states. running contains five child states: filling, heating, holding, draining, complete. The machine is always in a specific child (e.g. running.heating), never in running directly.
  • Inherited transitions. ABORT and E-STOP are declared once on running and are available from every child state. No duplication.
  • Compound state chrome. The ABORT/E-STOP buttons and the batch info bar are inside running but outside any child state. They stay visible across all phase transitions.
  • Entry/exit hooks. running has <mn-init> (records start time) and <mn-exit> (calculates run duration). These fire when entering/leaving the compound state, not on internal child transitions.
  • Structural transitions. Guards and actions are child elements of <mn-transition>, not s-expressions in attribute strings.
  • ISA-88 compliance. The abort transition is available from every operational phase, as required by the ISA-88 batch control standard.
