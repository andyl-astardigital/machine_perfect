Lesson 1

States and transitions

A machine has states. Only one is active. mp-to declares transitions. Click NEXT to cycle the traffic light.

<div mp="traffic">
  <div mp-state="red">
    <button mp-to="green">NEXT</button>
  </div>
  <div mp-state="green">
    <button mp-to="yellow">NEXT</button>
  </div>
  <div mp-state="yellow">
    <button mp-to="red">NEXT</button>
  </div>
</div>

No JavaScript. mp creates the machine. mp-state declares states. mp-to declares transitions. The HTML IS the state chart.

Lesson 2

Context, s-expressions, guards

mp-ctx holds data. mp-text displays it. (inc! n) modifies it. mp-guard blocks transitions conditionally.

0

Reset to zero?

<div mp="counter" mp-ctx='{"count": 0}'>
  <p mp-text="count">0</p>
  <button mp-to="." mp-action="(inc! count)">+1</button>
  <button mp-to="confirm" mp-guard="(!= count 0)">
  • mp-to="." — self-transition: stay in state, run the action
  • (inc! count) — the ! means mutation
  • mp-guard — blocks when falsy. Try resetting at zero

Lesson 4

Components: define once, use many

<template mp-define> defines a reusable machine. Each mp instance has its own state. Flip one card — the others don't move.

<template mp-define="flip-card">
  <div mp-state="front">...<button mp-to="back">Flip</button></div>
  <div mp-state="back">...<button mp-to="front">Back</button></div>
</template>
<div mp="flip-card" mp-ctx='{"label":"A","secret":"🎉"}'></div>
<div mp="flip-card" mp-ctx='{"label":"B","secret":"🚀"}'></div>

Lesson 7

Lists with mp-each

mp-each repeats a template for every item in an array. mp-key gives each item a stable identity for efficient updates. Type an item and press Add.

No items yet.

<template mp-each="items" mp-key="name">
  <li>
    <span mp-text="name"></span>
    <button mp-action="(remove-where! items :name name)">
  </li>
</template>
  • mp-each — renders the template for each item in the array
  • mp-key — enables efficient add/remove without re-rendering everything
  • Inside the template, item properties (name) are directly accessible
Lesson 8

Temporal behavior

(after ms state) transitions after a delay. (every ms expr) runs on an interval. Both are scoped to the state — enter starts them, exit stops them.

5

5

0

<div mp-state="running"
     mp-transition="(do
       (every 1000 (dec! remaining))
       (after 5000 done))">
  • (every 1000 expr) — runs every second while in this state
  • (after 5000 done) — transitions to "done" after 5 seconds
  • Leave the state? Timers are automatically cleaned up. No manual teardown
Lesson 9

Global stores

<mp-store> declares shared state. Any machine reads it via $store.name. Any machine writes it via set!. Change the color below — both machines see it.

Settings machine

Display machine

Current color: 

<mp-store name="prefs" value='{"color":"blue"}'></mp-store>
<button mp-to="." mp-action="(set! $store.prefs.color 'red')">
<span mp-text="$store.prefs.color"></span>

$store is shared across all machines. Write with set!, read with $store.name.field. No import, no wiring — just a name.

Lesson 10

Slots — component content projection

<mp-slot> is a placeholder in a template. Callers inject content. Named slots target specific positions. Same component, different content.

Warning

Are you sure you want to proceed?

Welcome

Thanks for trying machine_perfect!

<template mp-define="dialog">
  <h3><mp-slot name="title">Default Title</mp-slot></h3>
  <div><mp-slot>Default content</mp-slot></div>
</template>
<div mp="dialog">
  <span slot="title">Warning</span>
  <p>Are you sure?</p>
</div>

Named slots (slot="title") target <mp-slot name="title">. Unnamed content fills the default <mp-slot>. If no content is provided, the slot's children render as fallback.

Lesson 11

Nested machines

Machines nest. Each child has its own independent state. Toggle an email below — the others don't move. The parent doesn't re-render when a child transitions.

Inbox

<template mp-define="email-item">
  <div mp-state="unread">...<button mp-to="read"></button></div>
  <div mp-state="read">...<button mp-to="unread"></button></div>
</template>
<div mp="email-item" mp-ctx='{"subject":"Welcome"}'></div>
<div mp="email-item" mp-ctx='{"subject":"Your first"}'></div>
<div mp="email-item" mp-ctx='{"subject":"Advanced"}'></div>

Machines are independent. Nested machines have their own state. The parent doesn't re-render when a child transitions. Each mp="email-item" is its own state machine.
